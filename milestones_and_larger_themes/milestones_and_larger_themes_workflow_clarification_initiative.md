# Management & Leadership in CS - Final Written Report

## Initiative: Milestones and Larger Themes Workflow Clarification

### Scope

This initiative defines a milestone-centered reporting procedure for HAAG teams so that weekly updates communicate more than raw activity. The procedure makes milestones the primary unit of progress review and requires each milestone to be tied to a larger theme. This allows project managers and advisors to see whether weekly work is advancing a broader strategic direction, rather than only recording that work happened.

The intended audience is primarily Project Managers, with student researchers contributing the milestone updates and computational advisors participating in monthly theme review. The procedure is designed to work inside HAAG's existing Slack and GitHub workflows rather than introducing a new tool. Its purpose is to improve consistency, comparability, and early detection of risk while keeping the reporting burden realistic for teams already managing research deliverables.

### Evidence of Alignment with HAAG's Goals

This initiative aligns with HAAG's goals because the current repository and operational materials consistently emphasize visibility, accountability, and early intervention in project tracking. Existing HAAG structures already rely on weekly reporting, PM synthesis, and milestone-oriented GitHub coordination. What was missing was a clear procedural layer explaining how milestone updates should be written, reviewed, escalated, and connected to larger strategic themes.

Observations documented throughout this project showed repeated issues with inconsistent status language, uneven proof of completion, and difficulty connecting weekly execution to longer-term priorities. Those problems directly affect HAAG's ability to monitor progress clearly across projects. By standardizing milestone reporting and adding a recurring larger-theme review, this initiative supports HAAG's goal of making progress tracking more transparent, comparable, and useful for intervention before deadlines are missed.

### Procedures Generated

This report contains one procedure: the **Milestones and Larger Themes Workflow Clarification Standard Operating Procedure**. The purpose of this SOP is to give any HAAG Project Manager a complete, reproducible guide for running weekly milestone reporting and monthly larger-theme review inside existing team workflows. It defines the required reporting fields, role responsibilities, review cadence, escalation rules, integration with GitHub milestone work, and success metrics.
