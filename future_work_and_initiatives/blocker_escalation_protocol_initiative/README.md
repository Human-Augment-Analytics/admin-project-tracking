# Structured Blocker Escalation Framework (HAAG)

## Overview

This repository contains the full documentation for the **Structured Blocker Escalation Framework** developed within the **Human Augmented Analytics Group (HAAG)** initiative.

The framework was originally piloted in the **R for Evolution** project and later generalized into a scalable, Project Manager-led escalation system designed for adoption across all HAAG projects.

The goal of this system is to improve the early identification, tracking, and resolution of project blockers, while maintaining a lightweight and practical workflow that integrates with existing reporting structures.

---

## Repository Purpose

This repository serves as a complete documentation package for:

- The implementation guide for Project Managers

Together, these documents explain both:
- **How the system was developed**, and  
- **How it should be implemented moving forward**

---

## Key Concept: Blocker Escalation Framework

The framework introduces a structured, time-based escalation model:

- **Week 1:** Blocker identified and monitored  
- **Week 2:** Recurring blockers are discussed and assigned an owner  
- **Week 3:** Persistent blockers are escalated to faculty-level support  

This system is designed to ensure that blockers do not remain unresolved across multiple reporting cycles.

## Design Principles

This framework is built on the following principles:

- **Lightweight implementation** (no additional tooling required)
- **Human-driven decision making**
- **Early intervention over reactive resolution**
- **Clear ownership of blockers**
- **Scalability across multiple HAAG projects**

## Origin

This framework originated from the **R for Evolution** project, where it was piloted in an environment without a CLEAR advisor. The Project Manager assumed operational responsibility for implementation, which informed the final design as a PM-led scalable system.
